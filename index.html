BOX model



Anotações da aula 560 - CSS e box-model

- A section meio que exige um heading dentro dela, senão fica dando warning no validador
- Sessão é um box (bloco) por padrão ele é full-width mas a altura não é pré definida
- A margin entre dois elementos não se soma, ela é tipo 'sobreposta', prevalecendo a maior
- Posso passar valores do padding/margin das seguintes formas:
  - os 4 lados com 10px = padding: 10px;
  - cima e baixo 15px e direita esquerda 10 = padding: 15px 10px;
  - cima 10px direita 5px baixo 15px esquerda 7px = padding: 10px 5px 15px 7px; 'sentido horário'
  - se eu deixar algum valor em branco ele atribui 0, assim se quero cima 10px baixo 10 direita e esquerda 0, posso passar assim = padding: 10px 0 10px;

Aula 579 - Propriedades especificas (largura, altura, padding, border e margin)


- Configuração de tamanho, cor de fundo etc não é aplicada se o display tiver como inline.
- O display block (só pra reforçar) é full-width por padrão.

-

Overflow

- Controla o comportamento de elementos que 'transbordam a lista' se eu colocar overflow: hidden; por exemplo, o exedente sera ocultado.
O overflow tem o controle especifico ainda sobre o eixo x (horizontal) e eixo y (vertical) (overflow-x ou overflow-y)
- Opção scroll - coloca uma barra de rolagem no eixo correspondente do excedente.
porem se não exceder ele continua com a barra de rolagem, para corrigir isso usamos
o overflow: auto; dai ele só coloca a barra se o conteudo exceder.

-

Responsividade

- Podemos controlar a largura com um width de 100% , e o max-widht de 1200 (tela de um pc grande), dai se a tela for maior que isso tbem ele para no 1200.
- para ajustar ao meio ao expandir, controlamos com a margin: 0 auto;
- Com o heigth é a mesma coisa.
- Se eu setar tanto o width/height e o max-width/height para 100% ele/s pegam a dimensão da viewport.
- Min-width/height funciona da mesma forma, porem com o min-height: o; por exemplo, se eu não tiver conteudo dentro da div ela não sera exibida, ja se eu setar um min-height ela aparecera (daquele tamanho) mesmo em branco (sem conteúdo).

-

box-sizing: content-box;

tudo relacionado a largura e altura vai ser tratado apenas NO CONTEÚDO, isso quer dizer que borda, margin, padding etc, serão SOMADOS ao conteúdo, conforme esta acontecendo com as bordas entre as divs abaixo.

-

box-sizing: padding-box;

- A diferença é que no padding-box ele é somado 'pra dentro'

-

box-sizing: border-box;

- Melhor forma de usar, tanto que ja setamos isso no geral (*{}) pois tudo que configurarmos (padding, border etc) vai se delimitar ao tamanho da div. só não entra a margin porque a margin é sempre 'pra fora' do conteúdo

-

Margin vs padding

- Só pra reforçar a margin 'empurra' o que ta ao redor (separa um elemento do outro) pois ela é externa e o padding 'empurra pra dentro' do box-model (do mesmo elemento).

Float

- Quer dizer que um elemento vai usar apenas o espaço dele e liberar o restante pra outros.
- As margens no display block entre 2 elementos se sobrepõe, ja no display float, elas usam e respeitam a de cada elemento (são somadas).
- Visualização da margem float -

- Quando usamos o float dentro de outros elementos (eles perdem o tamanho).
- Quando o alinhamento dos elementos é float: left; e sobre espaço a direita (final do range) ficara um espaço nessa direita, dai se colocarmos um outro elemento logo apos os que estão usando o float, ele ocupará esse espaço. Para resolver isso usamos o atributo

clear:

left; no caso pq o float é left tbem, esse novo elemento vai pra fora do float.
posso usar o clear: right tbem (caso o float seja right), ou simplesmente

clear: both;

que vai limpar tanto esquerda como direita, independente de onde o float esteja.
- Posso fazer esse 'clear' direto no final da div com o pseudo-elemento after, segue exemplo na div class = grid abaixo.

Cores HTML



Cores in w3Schools

seletor de cores HTML in w3Schools.

- Podemos setar a cor pelo nome dela, por valor hexadecimal, ou rgba.
- Hexadecimal #FFFFFF significa que os dois primeiros FF são vermelho, os próximos 2 verde e os 2 ultimos azul. FF indica que ele ta ligado e 00 desligado. podemos representar com apenas 3 0´s ou 3 f´s, (#0F0 - indica que só o azul está ligado #00FF00). pois caso tenhamos um par de f ou 0 podemos representar com apenas 1 (respeitando o limite de 3 caracteres).
- O hexadecimal é calculado com a seguinte 'régua': ABCDEF123456789.
- A regua do rgba (onde a é sombra (opacidade), logo não é obrigatório usar) vai de 0 a 255 (isso pra cada valor (r-vermelho, g-verde, b-blue)). no hexadecimal essa opacidade é adicionada somando-se mais um par de valores ao final (ficando com 8 #FFFFFF00)
- A opacidade no rgba é representada por porcentagem (de 0 a 1), logo 0.75 é igual a 75%.

Position



Position in MDN

-

Relative:

corresponde a sua posição em relação aos outros elementos, qualquer 'distancia' de top ou left que colocarmos, ele irá se afastar do elemento mais próximo a ele.

-

Absolute:

Já a absolute é relativa a view port, ou a div/elemento que esta 'em volta dela'. Ela (div ou elemento) deixa de ocupar aquele espaço (que é prontamente assumido por outra div/elemento) e fica meio que 'flutuando' na página, ficando por cima de qualquer elemento que tenha o mesmo posicionamento que ela.
- Caso eu queira posicionar algum elemento/div dentro do outro, posso colocar:
div#interna {
    position: absolute;
    top: 50%;
    left: 50%;
- porem assim a 'ponta' do elemento que ficaria no meio, e todo seu espaço a direita iria 'descentralizar', para ajustar isso, vemos o tamanho do elemento e 'tiramos a metade' dele nas margins, ficando assim um elemento de 300px por ex.:
    margin-top: -150px;
    margin-left: -150px;
}
- Exemplo na div vermelha abaixo, e sua div interna 'branca'. A branca tem 100px w/h, alinhamento top e left de 50% e margin t/l de 50px (metade do tamanho dela).
- O posicionamento é definido no css com os atributos:

top, botton, right e leftt

, Como se fosse a magin.

-

Z-index:

Define quem vai ficar por cima de quem, quanto maior o valor maior a 'prioridade' de ficar 'por cima', tbem podemos colocar o z-index como -1, assim ele vai 'pra trás' dos outros elementos.

-

Fixed:

O elemento fica 'preso/fixo' onde a gente mandar, respeitando o top, botton, left etc que setarmos. ex div rubro-negra que vai pro topo.:

-

Sticky:

Deixa as coisas 'presas' onde vc mandar, geralmente com base no atributo top: 0; Exemplo no menuSuperior.

BreakPoints in MDN (set a breakpoint)

Breakpoints:

è o tamanho da tela do dispositivo, qndo eu quero saber o tamanho da tela para tratar o comportamento dos elementos com relação a ela.

Media Query:

Coloco uma regra no css que se a tela chegar naquele tamanho a div vai ter um comportamento. bem simples:
@media (max-widht: 320px {
    .break .colBreak {
    width: 100%;
    }
}

- Um modo menos trabalhoso de se fazer isso é setar o @media p/ 800px (tamanho de tablet) dai tudo que for menor que 800px (tablets e celulares) terão o comportamento dentro do
@media (max-widht: 800px) {
    .break .colBreak {
    width: 100%;
    }
}

- para ter outro comportamento entre os tamanhos (tablet por exemplo) usamos o and.
@media (min-width: 321px) and (max-width: 768px) {
    .break .colBreak {
    width: 100%;
    }
}
Os quadrados verdes tem 3 exemplos de tamanhos no css: desktop, tablet e celular.

Flexbox in MDN

Guia CSS-Tricks

Anotações aula 589:

- Flex box =

Caixa Flexivel.



- Flex box é tipo uma 'manipulação' de caixas similar ao float do css porem direto no html

- Imagem referencia dos 'Flex-itens' abaixo.

- Nós 'ativamos' o flex-box definindo a propriedade de uma div (que é display block por padrão), para

display: flex;

. Qunado setamos isso, nós passamos a controlar o comportamento dos filhos da div.

-

flex-direction: row ou column;

Define a direção do eixo principal (main axis) e perpendicular (cross axis). E tipo como se fosse uma manipulação do display block e line (porem aqui ele sempre vai ser full-width)

-

Justify-content:

Alinha os itens internos no eixo principal (main axis).
    : flex-start (esquerda pra direita)
    : flex-end (direita para esquerda)
    : space-around (deixa um espaço entre eles na direita e esquerda, tipo um padding 0px 20%)
    : space-between (espaço entre cada item igualmente 'colando' o primeiro a esquerda e o ultimo a direita)
    : space-evenly (similar ao space-around porem distribui o espaço igualitariamente).

-

align-items:

Alinha os itens internos no eixo perpendicular (cross axis).
     Também temos as propridades de alinhamento nas laterais, topo e botton e center.
    : baseline (Alinha delimitando uma linha na fonte dos itens internos, mesmo que um seja maior que o outro.)

-

align-content:

Alinha o conteúdo dos itens.

-

flex-wrap: wrap;

Impede que um elemento sobreponha o outro ao diminuir a tela 'quebrando a linha' do elemento ao ficar muito pequeno.

- Para alinhar um elemento exatamente ao centro da tela, setamos as duas propriedades: (Justify-content e align-items como center).

- Quase todas as propriedades de alinhamento tem as mesmas opções (flex-start, flex-end, center space-around space-between etc).
